AutoFlow — Lista de avance y pendientes
Fecha: 10/02/2026

RESUMEN DE LO QUE YA EXISTE

1) Estructura base MVC
- Controladores: Auth, Home, Maintenance, Rental, Vehicle.
- Modelos: BaseModel, Maintenance, Rental, User, Vehicle.
- Vistas: Auth, Maintenance, Rentals, Vehicles, Public.
- Enrutado: Router/web.php.
- Núcleo: Core/View.php.
- Entrada pública: Public/index.php.

2) Frontend inicial
- CSS global: Public/Css/Global.css.
- JS por pantalla: rentals-form.js, vehicle-form.js, vehicle-show.js.
- Carpeta de imágenes y uploads para vehículos.


PENDIENTES

A. Revisión técnica y saneamiento
1. Revisar rutas definidas y coherencia entre Router y Controladores.
2. Validar que todos los métodos referidos en rutas existan en controladores.
3. Verificar que cada vista tenga su controlador y modelo asociado.
4. Revisar Core/View.php (carga de vistas, layouts, errores 404/500).
5. Definir convenciones de naming y estructura de carpetas.

B. Base de datos
1. Definir esquema completo (tablas, claves, índices).
2. Crear migraciones/SQL inicial.
3. Ajustar Database.php para conexión segura y configurable.
4. Asegurar uso consistente de prepared statements.

C. Autenticación y usuarios
1. Registro con validaciones (email, password fuerte, duplicados).
2. Login con sesiones seguras.
3. Logout.
4. Roles/permisos (admin/cliente/operador).
5. Protección de rutas y vistas.

D. Módulo de Vehículos
1. CRUD completo (crear, listar, editar, eliminar).
2. Subida/gestión de imágenes (validar tipo/tamaño, borrar al eliminar).
3. Filtros y búsqueda (marca, modelo, año, disponibilidad).
4. Detalle público de vehículo (share).

E. Módulo de Rentals (alquileres)
1. Crear alquiler con validación de fechas y disponibilidad.
2. Cálculo de precio (días, tarifas, extras).
3. Flujo de estados (pendiente, activo, finalizado, cancelado).
4. Historial de alquileres por usuario.

F. Módulo de Maintenance (mantenimiento)
1. Registro de mantenimiento por vehículo.
2. Planificación/estado (pendiente/en proceso/completado).
3. Historial y costos.

G. Validaciones y seguridad
1. Validaciones server-side en todos los formularios.
2. Sanitización de inputs.
3. Manejo de errores y mensajes al usuario.
4. CSRF tokens en formularios críticos.

H. UI/UX
1. Unificar layout (Header/Footer) en todas las vistas.
2. Mejorar formularios (errores visibles, campos requeridos).
3. Página de error 404 amigable.
4. Responsive y accesibilidad básica.

I. Testing y QA
1. Pruebas manuales de rutas críticas.
2. Casos borde (fechas inválidas, inventario agotado, uploads).
3. Validación de permisos.

J. Documentación
1. README con setup (XAMPP, DB, rutas).
2. Manual rápido de uso (admin y cliente).
3. Guía de despliegue.


NOTAS DE ORGANIZACIÓN
- Prioridad inmediata: B (Base de datos) + C (Auth) + D/E (vehículos/alquileres).
- Luego: F (mantenimiento) + G (seguridad) + H (UI/UX).
- Cerrar con: I (QA) + J (documentación).
